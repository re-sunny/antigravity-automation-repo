# Backend Coding Style & Rules

이 프로젝트의 모든 백엔드 코드는 아래 제약 사항을 무조건 따릅니다.

## 1. 코딩 스타일 및 구조
- 번호 매기기 방식(첫째, 둘째) 대신 반드시 대시(`-`)를 사용하여 가독성을 높입니다.
- 함수와 클래스에는 목적, 입력 인자, 반환 값의 구조를 명시한 `docstring`을 필수로 작성합니다.
- 예외 처리는 에러가 발생한 단계(`step`)와 상세 원인(`traceback`)을 파악할 수 있도록 방어적으로 짭니다.

## 2. 보안 및 파일 제약 (Activation: Always On)
- 에이전트는 프로젝트 루트 외부의 시스템 파일에 접근하거나 쓰기 작업을 수행할 수 없습니다.
- 터미널 명령어(`run_command`)를 제안할 때는 반드시 사전에 목적을 명시하고 사용자 승인을 구해야 합니다.
